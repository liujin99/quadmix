> 2026-06-11 | 基于 BMK-v2 效果不佳的深度复盘与重新设计

| 缺少 lambada | 文学风格敏感，信号质量中 |

## 二、分析框架

验证集是 proxy 回归模型的 target。加入一个 proxy 无法有效预测的任务 = 给 target 注入噪声 = 回归模型学得更差（论文 Section 6, line 944-966）。

### 两个核心评估维度

| 维度 | 含义 | 为什么重要 |
|------|------|-----------|
| **1M 可学性** | 1M proxy 能否学到文本分布（非解题） | 学不到 → loss 接近随机 → 噪声 |
| **分布敏感度** | 不同训练数据是否产生显著不同的 loss | 不敏感 → LightGBM 无信号可学 |

**最高准则**：信号质量 = 1M proxy 在不同训练混合下的 loss 区分度。

### 关于 N（样本量）的澄清

val_loss 计算为文档级平均：`val_loss = mean over ALL docs of (per-doc loss)`。

每个 doc 权重 = 1/N_total。因此 N 小的任务天然被降权——其 loss 不稳定性对整体 val_loss 的影响与其占比成正比。例如 copa (N=100) 在 10,000 docs 中仅占 1%，即使 loss 方差大，对整体影响也仅 1%。**N 不是独立问题，信号质量才是决定因素。**

### 1M Proxy 模型能力边界

1M 参数模型（2 层 transformer, 256 dim, SwiGLU+RMSNorm）：

| 能力层级 | 1M 能学到？ | 举例 |
|---------|-----------|------|
| 词频/双词统计 | ✅ | "the" 后接 "cat" 的概率 |
| 简单句法模式 | ✅ | 主谓宾结构 |
| 主题/域分布 | ✅ | 科学文本 vs 叙事文本的词汇差异 |
| 表面文本风格 | ✅ | 新闻风格 vs 对话风格 vs 学术风格 |
| 多步推理 | ❌ | ARC-C 的因果推理链 |
| 深度阅读理解 | ❌ | BoolQ 的段落-问题关系 |
| 长程依赖 | ❌ | LAMBADA 的跨段末词预测 |
| 知识回忆 | ❌ | Jeopardy/MMLU 的具体事实 |

**关键洞察**：即使 1M 不能"解题"，只要任务文本的**词汇/主题/风格分布**对训练数据敏感，loss 就有区分力。但如果任务文本完全超出 1M 的理解能力（如深度阅读理解），loss 接近随机，成为噪声。

## 四、Ans% 不是选择标准（关键结论）

| 任务 | Ans% | v3 入选? | 原因 |
|------|------|---------|------|
| piqa | 59.7% | ✅ | 1M 可学（简单物理场景） |
| hellaswag | 37.5% | ✅ | 1M 可学（叙事续写） |
| winogrande | 22.5% | ✅ | 1M 部分可学（句子结构） |
| arc_challenge | 19.2% | ✅ | 1M 部分可学（科学文本） |
| arc_easy | 16.9% | ✅ | 1M 可学（简单科学） |
| lambada | 1.8% | ✅ | 1M 部分可学（段落分布） |

## 五、v3 设计

### 选择原则

1. **1M 可学性**（最高）：proxy 能学到文本分布，loss 非随机
2. **分布敏感度**：不同训练数据产生显著不同的 loss

### 入选任务（10 个）

| 任务 | N | Cap | 信号 | 入选理由 |
|------|---|-----|------|---------|
| hellaswag_zeroshot | 10,042 | 2000 | 强 | 叙事续写 = 标准 LM 任务，论文验证过的最佳 proxy 任务 |
| arc_easy | 2,376 | 2000 | 强 | 简单科学 QA，词汇/主题分布可学 |
| piqa | 1,838 | 1838 | 中强 | 物理场景短而直观，答案文本长 (~98c) 提供分布信号 |
| lambada | ~5K | 2000 | 中 | 段落文本分布可学（文学/叙事风格敏感），即使末词预测不准 |
| arc_challenge | 1,172 | 1172 | 中 | 科学文本分布可学（即使推理部分超出 1M） |
| winogrande | 1,267 | 1267 | 中 | 句子结构简单，代词模式有一定可学性 |
| winograd | 273 | 273 | 中 | 同 winogrande，N 小但仅占 2.3% 权重 |
| copa | 100 | 100 | 中 | 因果场景简单直观，N 小但仅占 0.8% 权重 |
| openbook_qa | 500 | 500 | 中 | 科学场景可学，N 小但仅占 4.3% 权重 |
| commonsense_qa | 1,221 | 1221 | 中弱 | 概念知识分布可学（边界情况，占 10% 权重） |

### 排除任务（11 个）及原因

| 排除任务 | 核心原因 |
|----------|---------|
| boolq | 1M 无法理解段落-问题关系，阅读理解超出能力 |
| squad | 阅读理解超出 1M，答案定位需要理解 |
| jeopardy | 需要具体知识回忆，1M 容量不够 |
| coqa | 对话阅读理解 + 答案极短 (0.6%) |
| bigbench_qa_wikidata | 答案极短 (7c)，简单事实查询，信号弱 |
| agi_eval_lsat_ar | 逻辑推理超出 1M |
| bigbench_dyck_languages | 符号任务，非自然语言 |
| bigbench_repeat_copy_logic | 符号任务 |
| bigbench_operators | 符号任务 |
| bigbench_cs_algorithms | 符号/算法任务 |
| bigbench_language_id | 语言分类，非自然语言建模 |

### v3 配置

| 参数 | 值 |
|------|---|
| Loss 策略 | Full-sequence（所有非 padding token 参与 loss） |
| 数据源 | eval bundle eval split |
| Cap | 2000 docs/task（N<2000 的任务用全部数据） |
| Tokenizer | GPT-NeoX-20B (vocab 50,432) |
| Block size | 2048 |

## 六、v1 / v2 / v3 对比

| 维度 | v1 (CORE-22tasks) | v2 (CORE-BMK) | **v3** |
|------|-------------------|---------------|--------|
| 任务数 | 21 | 10 | **10** |
| Loss 策略 | continuation-only | full-sequence | **full-sequence** |
| 1M 可学任务占比 | ~5/21 (24%) | 3/10 (30%) | **10/10 (100%)** |
| 噪声任务数 | 16 | 7 | **0** |
| 最大单任务占比 | 10.8% (cap-5000) | 54% (bigbench_qa) | **17% (hellaswag)** |
| 预计 docs | 46,926 | 37,600 | **~11,671** |
| 信号纯度 | 低 | 低 | **高** |
| 类别覆盖 | 5/5 | 3/5 | **4/5** (缺符号推理*) |

*符号推理类别中所有任务均为非自然语言（括号序列、自定义运算符、算法输出），1M proxy 无法从中获取任何自然语言分布信号，排除合理。

## 七、论文 Ablation 证据支持

论文 Section 6 (line 944-966) 为每个 benchmark 单独训练回归模型：
> "HellaSwag has larger variance than others... making the loss on it harder to predict"

这说明即使是论文 5 个 BMK 任务中，HellaSwag 的 proxy 预测也是边界情况。我们的 v3 保留 HellaSwag（信号最强）+ 添加更多 proxy-friendly 任务，而非添加更多难以预测的任务。
