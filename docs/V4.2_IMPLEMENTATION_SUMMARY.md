## Key Features

### 1. Backward Compatibility
- Validation sets without `task_labels` (e.g., openhermes-10k) work exactly as before
- `ProxyResult` with `per_task_losses=None` behaves identically to v4.1
- Optimizer falls back to aggregate model when per-task losses are unavailable

### 2. Discrimination-Adaptive Weighting
- **Formula**: `weight[task] = std[task] / sum(std[all_tasks])`
- **Rationale**: Standard deviation normalization balances high and low discrimination tasks
- **Example**: Task with 10x variance gets ~3.16x weight (not 10x)

### 3. Zero-Variance Handling
- Tasks with variance < 1e-8 are automatically excluded
- Warning message printed when tasks are excluded
- Remaining tasks are re-normalized

### 4. Diagnostic Output
- Per-task R² scores for model quality assessment
- Per-task weights for understanding task importance
- Per-task standard deviations for understanding discrimination power
