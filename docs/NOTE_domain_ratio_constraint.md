# QuaDMix 域配比结构性约束分析

> 2026-06-29 | 发现：QuaDMix 搜索能产生大幅域分布变化，但集中在 3 个大域的配比上，7 个小域始终边缘化

## 1. 问题

QuaDMix-BMK 中训效果不够好。怀疑：搜索出来的"最优配比"是否真正改变了域分布？如果原始数据集由少数大域主导数据量，采样机制是否天然限制了域比例的可调范围？

## 2. 机制分析

### 2.1 采样公式

```
S(r̄) = (2/(1+e^{-λ(ω-r̄)}))^η + ε,  if r̄ <= ω
S(r̄) = ε,                             if r̄ > ω
```

- S_max ≈ 2^η + ε ≤ **2.001**（最多重复 2 次）
- S_min = ε ≈ 0.001（几乎不选）
- 对每个域独立计算，参数 (λ, ω, η, ε) 按域配置

### 2.1.1 ω 参数的硬约束（论文设定）

```python
# param_sampler.py:71,81 + types.py:217-219
omega_raw = self._rng.uniform(0.0, 1.0)  # [0, 1]
rescaled_omega = omega_raw * 0.1          # → [0, 0.1]
```

**ω 的实际范围是 [0, 0.1]**，即只有质量排名前 10% 的文档才能获得质量加权提升（S > ε）。

- rank ≤ ω（前 10%）→ 走 sigmoid 分支，S ∈ [ε, 2.001]
- rank > ω（后 90%）→ S = ε ≈ 0.001，几乎不选

这意味着每个域的**有效文档池被压缩到前 10%**。

### 2.2 搜索空间

搜索空间包含：
1. **质量权重** `domain_weights`：控制每个域内如何评分质量
2. **采样参数** `(λ, ω, η, ε)`：控制每个域内的采样曲线形状

**关键发现：没有任何参数直接控制"最终数据集中域 A 占 X%，域 B 占 Y%"。**

最终域分布是以下公式的副产品：

```
域 m 最终 token 数 = Σ_{doc ∈ m} S(r̄_doc) × tokens_doc
```

### 2.3 双重约束叠加

| 约束 | 效果 |
|------|------|
| **S_max ≤ 2.001** | 单文档最多重复 2 次，单次 pipeline 的域间采样率比仅 ~2.7x |
| **ω ≤ 0.1** | 每个域只有前 10% 文档有资格被"选中"，其余 90% 几乎被丢弃 |

**注意**：单次 pipeline 的采样率变化小（2.7x），但 874 组实验累积的域占比变化很大（见 3.3 节）。问题不在变化幅度，而在变化集中在大域。

### 2.4 理论约束（考虑 ω 限制后）

| 域 | 原始文档数 | 前 10% 有效池 | 占比 | 最大放大 (S=2) | 理论最大占比 |
|----|-----------|-------------|------|--------------|------------|
| D0 (Industrial) | 6.66M | 666K | 41.1% | 1.33M | ~48% |
| D1 (Social) | 3.56M | 356K | 22.0% | 712K | ~26% |
| D9 (Arts) | 2.98M | 298K | 18.4% | 596K | ~22% |
| D7 (General) | 1.41M | 141K | 8.7% | 282K | ~10% |
| D6 (History) | 455K | 45.5K | 2.8% | 91K | ~3% |
| D2 (Science) | 375K | 37.5K | 2.3% | 75K | ~3% |
| D5 (Literature) | 204K | 20.4K | 1.3% | 41K | ~1.5% |
| D3 (Religion) | 297K | 29.7K | 1.8% | 59K | ~2% |

**ω 约束的影响**：
1. 每个域的有效文档池被压缩到 10% → 小域（如 Language 83K）的有效池只有 ~8.3K
2. 大域的前 10% 仍然远大于小域的前 10% → Industrial 前 10% = 666K，Language 前 10% = 8.3K，比例 80:1
3. 搜索只能在各域的前 10% 中做选择，无法利用后 90% 的数据

理论上搜索可以让大域缩到几乎 0，小域放大 2 倍。但实际上采样曲线是 sigmoid 形状，大部分文档的 S 值在 0.1-1.5 之间，搜索能调的只是曲线形状，不能对每个文档独立控制。

## 4. 结论

### 4.1 核心问题（修正）

**QuaDMix 的搜索空间能产生大幅域分布变化，但集中在 3 个大域的配比上。**

874 组实验的域分布变化很大（Industrial 0.8%-83%，Social 0.5%-79%），但 PCA 显示 97.4% 的方差集中在前 3 个主成分，本质上只在 Industrial/Social/Arts/General 四个大域之间做 tradeoff。7 个小域（Science, History, Religion, Literature, Philosophy, Philology）始终边缘化（每个 <5%），搜索从未真正探索过它们的配比空间。

### 4.2 根因

1. **原始分布极端偏斜**：Top-3 域（Industrial 41%, Social 22%, Arts 18%）占 81%，小域绝对数量太少
2. **采样机制的放大效应**：S_max ≤ 2.001 意味着任何域最多被放大 2 倍。Industrial 41% × 2 = 82%，Philology 0.5% × 2 = 1%，绝对差距从 40.5pp 扩大到 81pp
3. **ω ≤ 0.1 硬约束**：每个域只有前 10% 文档参与采样。Industrial 前 10% = 666K，Philology 前 10% = 8.3K，比例 80:1。搜索只能在各域的前 10% 中做选择
4. **无显式域权重参数**：搜索空间只有质量权重和采样曲线参数，没有直接控制域比例的参数

### 4.3 影响

- 搜索的"最优"是在**大域配比空间**内的局部最优，小域配比从未被有效探索
- 如果最优配比需要显著提升小域比例（如 Science 从 3% 提到 15%），QuaDMix 做不到——小域占比被原始分布的绝对数量锁死
- **有效搜索空间是 3 维而非 10 维**：搜索本质上只在 Industrial/Social/Arts 之间做 tradeoff
- LightGBM 可能学到了大域配比的信号，但对小域配比的预测能力为零

### 4.4 核心洞察：域分布变化大，但维度低

**问题本质**：原始分布极端不平衡 → 搜索只能在大域之间做 tradeoff → 小域始终边缘化 → 如果最优配比需要提升小域，QuaDMix 无法到达

```
因果链：
原始分布偏斜 (Top-3 占 81%)
    ↓
采样约束 (ω≤0.1, S_max≤2) 放大绝对差距
    ↓
874 组实验的域分布变化大，但集中在 3 维子空间
    ↓
7 个小域始终 <5%，从未被有效探索
    ↓
如果最优配比需要小域占比 >10%，QuaDMix 无法到达
    ↓
中训效果受限
```

**关键矛盾**：
- **域分布变化大**：Industrial 0.8%-83%，Social 0.5%-79%（不是之前假设的 <1%）
- **但维度低**：97.4% 方差集中在 PC1+PC2+PC3，本质上只在 3 个大域之间做 tradeoff
- **小域被锁死**：7 个小域合计 <15%，每个 <5%，搜索从未真正探索过它们的配比空间

**与论文的对比**：
- **RefinedWeb (论文)**：26 域，分布 1%-9%，接近均匀。每个域都有足够的数据量被搜索探索，有效搜索空间接近 26 维。
- **Essential-Web (我们)**：10 域，分布 0.5%-41%，极度偏斜。搜索只能在大域之间做 tradeoff，有效搜索空间退化为 3 维。

**结论**：
问题不在"域分布变化小"，而在"域分布变化集中在大域，小域被锁死"。改进方向是**提升小域的有效数据量**（预处理均衡化或放宽采样约束），而不是加实验数或换回归模型。

## 5. 修复方向

### 5.1 预处理阶段均衡化（推荐）

在 QuaDMix 之前，先按目标比例对各域进行预采样（如均匀分布或 Chinchilla 比例），再用 QuaDMix 做域内质量筛选。

**优点**：不改 QuaDMix 代码，只改输入数据。原始分布从 82x 偏斜降到 ~10x，搜索的有效维度从 3 维扩展到 7-10 维。

**缺点**：大域丢失大量文档（如 Industrial 从 6.6M 降到 ~1.6M），客户可能不接受"浪费"数据。

### 5.2 放宽 ω 约束

当前 ω ∈ [0, 0.1]（前 10%），可尝试：
- ω ∈ [0, 0.3]（前 30%）：扩大有效池 3 倍
- ω ∈ [0, 0.5]（前 50%）：扩大有效池 5 倍
- ω ∈ [0, 1.0]（全部）：所有文档都有机会被选中

**权衡**：ω 越大，质量筛选越弱，但域比例可调范围越大。对比 quality-top-k 基线可能有质量劣势。

### 5.3 引入显式域权重

在采样公式中加入域级权重参数：

```
最终采样值 = domain_weight[m] × S(r̄)
```

其中 `domain_weight[m]` 是搜索空间中的显式参数（如 Dirichlet 分布采样），直接控制域间比例。

**问题**：S 的采样逻辑在 S=1 附近有概率跳变（S≥1 确定性采样，S<1 概率采样），乘一个 domain_weight 会在 S=1 附近造成不连续的跳变，搜索优化器难以处理。

### 5.4 两阶段搜索

1. **阶段 1**：搜索最优域配比（低维，如 10D simplex）
2. **阶段 2**：在固定域配比下，搜索最优质量采样参数

**优点**：解耦域配比和质量筛选，避免 S=1 跳变问题。

**缺点**：改算法，增加搜索参数（10 个域配比），需要更多实验次数。

### 5.5 与 L2 映射的关系

L2 映射（10→26 类）能部分缓解问题：
- 分布偏斜度降低：D0 (41%) 拆成 ~4 个 L2 子域，最大域占比从 41% 降到 ~10-15%
- 更接近 RefinedWeb 的 26 域结构（论文 max 9%, min 1%）

**但不能根治**：
- 采样机制不变：ω ≤ 0.1, S_max ≤ 2.001 仍然硬约束
- 子域内部仍然偏斜：拆分后某些 L2 域可能只有几千文档，有效池更小
- 根本矛盾仍在：小域被原始分布的绝对数量锁死

**结论**：L2 映射是**必要但不充分**的改进。推荐组合：**L2 映射 + 预处理均衡化**，这样既有细粒度域标签，又有足够的数据集组成变化范围。
